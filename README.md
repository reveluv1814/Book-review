# Prueba Técnica - Book Review App

Aplicación web de Review de Libros. Parte de la prueba técnica para la posición de Desarrollador Junior.

## Stack Tecnológico

### Backend

- Next.js API Routes con TypeScript
- PostgreSQL - Base de datos relacional
- jsonwebtoken 9.0.3 - Autenticación JWT
- bcryptjs 3.0.3 - Hashing de contraseñas

### Frontend

- Next.js 16.2.2 - App Router con TypeScript
- React 19.2.4 - Librería de interfaz de usuario
- Tailwind CSS 4 - Utilidades CSS y custom color system
- React Hook Form 7.72.0 - Gestión de formularios
- Zod 4.3.6 - Validación de schemas
- react-toastify 11.0.5 - Toasts
- Remix Icons - Librería de iconos (CDN)

### Autenticación

- JWT (JSON Web Tokens)
- Cookies httpOnly

### Testing

- Playwright - Pruebas e2e

### Dev Dependencies

- ESLint - Linting de código
- Prettier - Formateador de código

## Mood Field

En cuanto el campo de "mood" o "sentimiento" en la aplicación de review de libros, se implementó como un input de tipo text, ya que puede servir como etiquetas o plabras clave. Esto permite a los usuarios describir lo que sintieron al leer el libro.

## Bugs o trade-offs

- No se implementó un sistema de roles para ls gestión de usuarios.
- No se implementó paginación y filtrado en la lista de reviews.
